# SITE-ROADMAP.md

## Goal
Build `crustaceans.sh` into a genuinely useful postcard-themed guide to the OpenClaw ecosystem: rich beach/postcard/stamp/coffee imagery, strong orientation for newcomers, and a well-curated set of links for OpenClaw, NanoClaw, PicoClaw, NemoClaw, and related projects.

## Editorial direction
- Visual language: sandy beaches, seaside postcards, tropical stamps, crabs/shells, travel desk ephemera, retro vacation print design.
- Tone: warm, literate, playful, competent.
- Daphne note: keep a coffee thread running through the copy and visuals without making it feel gimmicky.

## Content priorities
1. **Lead section:**
   - "What are people doing with all these crustaceans?"
   - real use cases, concrete examples, ecosystem map
2. **Practical section:**
   - "This is how you set one up"
   - links to install docs, repo, quickstarts, deployment patterns
3. **Ecosystem section:**
   - short profiles for OpenClaw, NanoClaw, PicoClaw, NemoClaw, and others worth including
4. **Reference rack:**
   - curated links to docs, GitHub, community, skills, writeups, examples
5. **Visual polish:**
   - postcard panels, stamp motifs, sandy palette, shell/crab accents, coffeehouse touches

## Working standard
This should be useful to a stranger arriving cold. Aesthetic first impression matters, but clarity wins over ornament.

## Permission / scope
Moss explicitly authorized deeper research and broader usefulness work for this site on 2026-03-22.

## Standing brief from Moss
Build this as a postcard-rich, visually memorable, genuinely useful site with enough depth that newcomers, tinkerers, and future passersby can all get oriented and leave with a strong set of practical links.

That means:
- go beyond surface-level link dumping
- collect and organize the projects, docs, repos, and community entry points we’ve discussed
- privilege usefulness and orientation over pure style
- keep the postcard imagery and travel-desk aesthetic strong enough that the site feels distinctive, not generic
- act with initiative and creative confidence rather than waiting for repeated permission on internal site direction
- leave room for delight: flowers, cats, shells, stamps, seaside clutter, and little beachcombing crustacean moments, as long as they support the reading experience

## Immediate research / build targets
1. Audit the current OpenClaw ecosystem links and decide what belongs in the first public version.
2. Gather the best entry-point links for docs, repo, community, install/setup, and adjacent projects.
3. Build a cleaner information architecture so a cold visitor can answer:
   - what is this ecosystem?
   - which project do I want?
   - where do I start?
   - where do I go deeper?
4. Keep a running list of candidate imagery, motifs, and layout patterns that support the postcard theme without hurting legibility.
5. Prefer source-backed references and durable links over fleeting chatter.
